  🎯 전체 워크플로우 개요

  Phase 1: 시스템 초기화 및 브라우저 설정

  1.1 Stealth 브라우저 생성
  - Playwright의 Chromium 엔진을 사용하여 브라우저 인스턴스 생성
  - headless=False로 설정하여 실제 브라우저 창을 띄워 사람처럼 보이게 함
  - 다음과 같은 Anti-Detection 파라미터들을 설정:
  --no-sandbox: 샌드박스 보안 비활성화
  --disable-setuid-sandbox: setuid 샌드박스 비활성화
  --disable-blink-features=AutomationControlled: 자동화 감지 비활성화
  --disable-web-security: 웹 보안 정책 비활성화

  1.2 브라우저 컨텍스트 설정
  - 뷰포트 크기를 1920x1080으로 설정하여 데스크톱 환경 시뮬레이션
  - User-Agent를 실제 Chrome 브라우저와 동일하게 설정:
  "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
  - JavaScript 초기화 스크립트 주입:
  Object.defineProperty(navigator, 'webdriver', {
      get: () => undefined,
  });
  - 이를 통해 navigator.webdriver 속성을 숨겨 자동화 도구임을 감춤

  Phase 2: 페이지 접근 및 기본 정보 입력

  2.1 KB생명 보험상품 페이지 로드
  - 대상 URL: https://www.kblife.co.kr/insurance-product/productDetails.do?linkCd=ON_PD_YG_01&productType=5
  - wait_until="networkidle" 옵션으로 모든 네트워크 요청 완료까지 대기
  - 3-5초의 랜덤 대기시간을 추가하여 자연스러운 사용자 행동 시뮬레이션

  2.2 생년월일 입력 자동화
  - generate_birthdate(age) 함수를 사용하여 현재 연도에서 나이를 뺀 생년월일 자동 생성
  - 월과 일은 랜덤하게 생성 (1-12월, 1-28일)
  - CSS 선택자 #birthday를 통해 생년월일 입력필드 탐색
  - page.fill() 메서드로 생년월일 입력 (예: "19961104")

  2.3 성별 선택 로직
  - 3단계의 fallback 메커니즘으로 성별 선택:

  - 방법 1: 기본 라디오 버튼 클릭
  const radios = document.querySelectorAll('input[name="genderCode"]');
  radio.value === "1" (남성) || "2" (여성)
  radio.checked = true;
  radio.click();

  - 방법 2: JavaScript 직접 실행
    - DOM 이벤트 직접 발생: change, input, click 이벤트
    - 부모 요소까지 이벤트 버블링 처리

  방법 3: Force 클릭
    - Playwright의 click(force=True) 메서드 사용
    - CSS나 JavaScript로 숨겨진 요소도 강제 클릭

  2.4 설계견적하기 버튼 실행
  - 페이지 내 모든 button 요소를 순회하면서 텍스트 내용 확인
  - "설계" 또는 "견적"이 포함된 버튼을 자동 감지 및 클릭
  - 버튼 클릭 후 페이지 전환 대기

  Phase 3: 페이지 안정화 및 동적 콘텐츠 대기

  3.1 고급 페이지 안정화 로직
  1. network idle 상태까지 대기 (최대 15초)
  2. 3초간 추가 대기 (동적 콘텐츠 로딩)
  3. 중요 요소들의 존재 확인:
     - input[type="checkbox"] (특약 체크박스들)
     - 보험료 표시 요소 (숫자+원 패턴)
  4. DOM readyState === 'complete' 확인
  5. 최종 2초 추가 대기
  6. 체크박스 개수 카운트 및 로깅

  3.2 모달 다이얼로그 자동 처리

  - "확인", "닫기", "취소" 버튼 자동 클릭

● ⚠️ 현재 시스템의 주요 문제점 및 해결방안

  문제점 1: 특약 활성화 100% 실패

  원인 분석:
  - KB생명 웹사이트의 특약들이 플랜 선택 이후에만 활성화되는 구조
  - 특약 체크박스들이 DOM에는 존재하지만 JavaScript로 비활성화된 상태
  - 보험료 계산이 서버 사이드에서 처리되어 즉시 반영 안됨

  해결방안:
  1. 플랜 우선 선택 로직
  # 1. 먼저 구체적인 플랜(종합든든, 종합실속 등)을 선택
  # 2. 플랜 선택 후 페이지 안정화 대기
  # 3. 그 다음 해당 플랜의 특약들 활성화 시도
  2. 서버 응답 대기 메커니즘
  # 특약 선택 후 AJAX 요청 완료까지 대기
  await page.wait_for_response(lambda response:
      "premium" in response.url or "calculate" in response.url)

  문제점 2: 플랜 정보 수집 실패

  원인 분석:
  - 플랜 라디오 버튼 감지 로직에서 0개만 발견
  - KB 사이트의 플랜 선택 UI가 예상과 다른 형태일 가능성

  해결방안:
  # 더 광범위한 플랜 선택자 탐색
  plan_selectors = [
      "input[name*='plan']", "input[name*='type']",
      "button[data-plan]", ".plan-selector",
      "input[value*='comprehensive']", "input[value*='basic']"
  ]

  - ❌ 보험기간/납입기간별 가격 (미구현)

  핵심 기술적 성취:
  1. 9단계 페이지 처리 파이프라인 구축
  2. 3단계 Fallback 메커니즘 (각 동작마다)
  3. 실시간 DOM 변화 감지 및 적응
  4. 완전 자동화된 휴먼라이크 동작
  5. 포괄적 에러 처리 및 복구 시스템

  이 시스템은 웹 스크래핑 자동화의 모든 핵심 요소를 포함하고 있으며, KB생명뿐만 아니라 다른 보험사 사이트에도 쉽게
  적용 가능한 확장 가능한 아키텍처를 가지고 있습니다.
